## 配置语言

在站点 `_config.yml` 中配置单一语言：

```yaml
language: zh-CN
```

配置多语言时，数组顺序同时表示默认语言和回退优先级：

```yaml
language:
  - zh-CN
  - en
  - ja
```

语言代码需要与主题 `languages/` 中的文件名匹配。Hexo 支持两字母 ISO-639-1 代码及其变体，例如 `en`、`zh-CN`、`zh-TW`。

## 语言文件

将 YAML 或 JSON 语言文件放在主题目录：

```text
themes/your-theme/
└── languages/
    ├── default.yml
    ├── en.yml
    ├── zh-CN.yml
    └── ja.yml
```

`default.yml` 可作为主题默认文案；实际语言仍由站点 `language`、页面 `lang` 和路径检测决定。

```yaml
# languages/zh-CN.yml
menu:
  home: 首页
  archives: 归档
  tags: 标签
  categories: 分类
  about: 关于

post:
  created: 发布于
  updated: 更新于
  read_more: 阅读更多

pagination:
  prev: 上一页
  next: 下一页

powered_by: 由 %s 驱动
article_count: 共 %d 篇文章
```

语言文件支持 `sprintf` 风格占位符。不要把用户输入直接作为格式字符串。

## 模板中使用

使用 `__()` 获取普通翻译：

```ejs
<%= __('menu.home') %>
<%= __('powered_by', 'Hexo') %>
```

使用 `_p()` 处理复数：

```yaml
# languages/en.yml
post:
  zero: No posts
  one: One post
  other: '%d posts'
```

```ejs
<%= _p('post', 0) %>
<%= _p('post', 1) %>
<%= _p('post', 5) %>
```

用于 HTML 属性时保留转义输出：

```ejs
<input type="search" placeholder="<%= __('search.placeholder') %>">
```

## 路径语言检测

`i18n_dir` 的默认值为 `:lang`：

```yaml
i18n_dir: :lang
```

Hexo 会把 URL 第一段与存在的语言文件匹配：

| 路径 | 检测结果 |
| --- | --- |
| `/index.html` | 默认语言，即 `language` 的第一项 |
| `/zh-CN/index.html` | `zh-CN`，前提是存在对应语言文件 |
| `/ja/index.html` | `ja`，前提是存在对应语言文件 |
| `/archives/index.html` | 默认语言；`archives` 不是语言代码 |

也可以在页面 Front-matter 中使用 `lang` 覆盖自动检测：

```yaml
---
title: About
lang: en
---
```

`i18n_dir` 只检测语言，不会自动翻译、复制或生成另一种语言的内容。

## 多语言内容结构

文章仍应放在 `source/_posts/` 下。使用语言变量组织文件和永久链接：

```yaml
# _config.yml
new_post_name: :lang/:title.md
permalink: :lang/:title/
```

```bash
hexo new "Hello World" --lang en
hexo new "你好世界" --lang zh-CN
```

可能生成以下结构：

```text
source/_posts/
├── en/
│   └── Hello-World.md
└── zh-CN/
    └── 你好世界.md
```

不要使用 `source/zh-CN/_posts/` 作为文章目录；Hexo 的文章处理约定仍以 `source/_posts/` 为根。

如果不同语言页面需要互相对应，建议在 Front-matter 中加入稳定的翻译标识或显式链接，例如：

```yaml
translation_key: getting-started
translations:
  zh-CN: /zh-CN/getting-started/
  en: /en/getting-started/
```

这些字段是主题自定义约定，不是 Hexo 内置配置。

## 语言切换器

先规范化默认语言，避免把 `config.language` 数组当成字符串：

```ejs
<%
const configuredLanguages = Array.isArray(config.language)
  ? config.language
  : [config.language || 'en'];
const defaultLanguage = configuredLanguages[0];
const currentLanguage = page.lang || page.language || defaultLanguage;
const translations = page.translations || {};
%>

<nav class="language-switcher" aria-label="Language">
  <% configuredLanguages.forEach(function(language){ %>
    <% const target = translations[language]; %>
    <% if (target) { %>
      <a href="<%- url_for(target) %>"
         hreflang="<%= language %>"
         class="<%= language === currentLanguage ? 'active' : '' %>">
        <%= language %>
      </a>
    <% } %>
  <% }) %>
</nav>
```

只有在目标翻译确实存在时才输出切换链接，不要仅通过字符串替换当前 URL 猜测翻译路径。
